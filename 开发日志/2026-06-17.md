## Bug 修复（同日）

### 1. `getChildDepartments(null)` SQL 问题
- **现象**：新增顶级部门时 `WHERE parent_id = NULL` 查不到数据，导致排序计算出错
- **修复**：判断 `parentId` 为 null 时使用 `WHERE parent_id IS NULL`

### 2. `createDepartment` 返回空对象
- **现象**：`last_insert_rowid()` 在 sql.js 中跨 `prepare` 调用失效，新建部门返回空 `id`
- **修复**：改用 `SELECT MAX(id)` 获取最后插入的 ID

### 3. 删除部门未检查关联用户
- **现象**：可直接删除有用户的部门，导致用户 `department_id` 变成悬空引用
- **修复**：`deleteDepartment` 增加用户计数检查，存在用户时抛出错误阻止删除；路由层返回 400 + 具体错误信息

### 4. 用户 API 增加 `department_id` 字段支持
- `POST /api/users` 创建用户时支持 `department_id`
- `PUT /api/users/:id` 修改用户时支持 `department_id`
- `db.createUser()` 和 `db.updateUser()` 同步支持

### 5. 用户管理页增加部门选择器
- 新建用户表单和修改用户弹窗中均增加 `TreeSelect` 部门选择器
- 部门数据从 `/api/departments/tree` 加载，以树状结构展示（与部门管理页一致）
- 支持搜索过滤部门节点
- 该字段为可选（`allowClear`），不强制选择
