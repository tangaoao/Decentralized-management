### 批量申请功能 — 多项目 + 多单号

**需求**：
- 场景 A：一个项目对应多个发放单号，单位可依次输入，列表中分条展示
- 场景 B：同一单位一次提交多个项目（同银行），列表中分条展示

### 强制修改密码功能

**需求**：管理员重置密码后，用户首次登录时强制设置新密码，需两次输入 + 密码强度校验。

**设计**：
- 数据库新增 `password_reset_required` 列（INTEGER DEFAULT 0）
- 管理员重置密码 → flag=1；用户改密成功 → flag=0
- 登录响应自动包含 `password_reset_required` 字段
- `ProtectedRoute` 检查 flag，拦截所有业务页面
- `/change-password` 不经过 ProtectedRoute，自行处理认证

**密码强度规则**：长度 ≥ 8 位 + 大写/小写/数字/特殊字符至少 3 种

**改动文件**：

| 文件 | 改动 |
|------|------|
| `server/src/db.js` | ALTER TABLE 新增 `password_reset_required` 列；`resetPassword()` 增加 `password_reset_required = 1`；新增 `changePassword(id, hash)` 函数 |
| `server/src/routes/auth.js` | 新增 `validatePasswordStrength()` 校验函数；新增 `POST /api/auth/change-password` 路由 |
| `client/src/api.js` | `authAPI` 新增 `changePassword()` |
| `client/src/auth/AuthContext.js` | 新增 `updateAuth()`、`getHomePath()`，暴露到 Context |
| `client/src/auth/LoginPage.js` | 登录后检测 `password_reset_required`，条件跳转 `/change-password` |
| `client/src/auth/ChangePasswordPage.js` | **新建** — 强制改密页面，含密码强度实时进度条、两次密码确认 |
| `client/src/layouts/ProtectedRoute.js` | 增加 `password_reset_required` 拦截，重定向到 `/change-password` |
| `client/src/App.js` | 新增 `/change-password` 路由 |
