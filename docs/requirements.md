# 项目需求文档：二次发放审批系统（三方协作）

## 1. 项目概述

- **项目名称**：二次发放申请审批系统
- **业务场景**：单位端发起"二次发放"申请，银行端审核，财政端全流程监管
- **核心价值**：实现二次发放申请的线上化流转、实时状态同步与统计报表分析
- **目标**：Web 应用，支持三类角色（财政、银行、单位）的独立视图与协作流程

## 2. 角色与权限矩阵

| 角色 | 核心职责 | 数据可见范围 | 可执行操作 |
|------|----------|-------------|-----------|
| 单位端 | 发起申请 | 仅本单位发起的申请 | 新增申请、查看列表、查看状态、修改被退回的申请 |
| 银行端 | 审核申请 | 所有待审核 / 已审核的申请 | 审核通过、退回修改 |
| 财政端 | 全流程监督 | 所有单位的全部申请 | 查看所有申请、汇总报表、穿透明细 |

## 3. 状态流转

```
单位提交 → [待银行审核 PENDING] ──银行通过──→ [已发放 ISSUED]（终态）
                                ──银行退回──→ [已退回 RETURNED]
                                                 │
                              单位修改重提 ────────┘（状态回到 PENDING）
```

## 4. 功能需求

### 4.1 单位端
- 新增申请（项目名称、发放单号、选择银行、申请理由）
- 查看本单位申请列表（状态筛选：全部/待审核/已发放/已退回）
- 修改被退回的申请并重新提交
- 5 秒轮询实时更新

### 4.2 银行端
- 待审核列表 Tab（status=PENDING）
- 已发放列表 Tab（status=ISSUED）
- 审核操作：通过发放（可选备注）/ 退回修改（必填退回原因）
- 5 秒轮询实时更新

### 4.3 财政端
- 汇总报表：按发放单号分组统计
- 全量申请列表（支持按单位、发放单号、状态筛选）
- 点击累计次数穿透明细
- 用户管理：查看/新增/删除用户（支持单位/银行/财政三种角色）

## 5. 验收标准

| 编号 | 场景 | 预期结果 |
|------|------|----------|
| AC-01 | 单位端提交申请 | 生成 PENDING 记录，银行/财政可见 |
| AC-02 | 银行端审核通过 | 状态变 ISSUED，三方同步 |
| AC-03 | 银行端退回 | 状态变 RETURNED，单位可见退回原因 |
| AC-04 | 单位端修改重提 | 状态变 PENDING，退回字段清空 |
| AC-05 | 财政端汇总报表 | 按发放单号分组，累计次数正确 |
| AC-06 | 财政端穿透明细 | 显示该发放单号下所有申请详情 |
| AC-07 | 多单位数据隔离 | unit1 看不到 unit2 的数据 |
| AC-08 | 同一发放单号多次申请 | 汇总次数与实际条数一致 |
| AC-09 | 银行数据隔离 | 银行用户仅能看到分配给自己的申请 |
| AC-10 | 银行越权操作 | 银行不能审核/退回分配给其他银行的申请 |
| AC-11 | 财政端用户管理 | 可新增不同角色用户，删除非己用户 |
